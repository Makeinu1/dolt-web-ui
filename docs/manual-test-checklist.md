# Dolt Web UI — ブラウザ手動試験チェックリスト

このドキュメントはブラウザで実施する最終受け入れ試験のチェックリストです。
E2E スクリプト（APIレベル）が通過した後、UI の動作・表示・インタラクションを人手で確認します。

**前提条件:**
- `./dist/dolt-web-ui -config config.yaml` でサーバ起動済み
- Dolt SQL Server（ポート3306）が起動済み
- `Test` データベースに複数テーブルとデータが存在する状態
- ブラウザは Chrome または Firefox（最新版）
- セクション7（バージョン比較）の実施前に Approve を **2回以上** 実施済みであること（比較用コミットが2つ以上必要）
- セクション9（コンフリクト解決）の実施前に意図的なコンフリクトを作成しておくこと（手順は各セクション冒頭に記載）

**テスト実行順序の注意:**
セクション1→2→3→...の順に実施してください。各セクション開始時点でワークブランチが存在することを前提とします（ブランチ削除テストはセクション16に集約）。

## セクション 9: コンフリクト解決

> **前提（コンフリクト作成手順）:**
> 1. ワークブランチ A でセル X を値「AAA」に変更 → Commit → Submit → Approve（main に反映）
> 2. 別のワークブランチ B で同じセル X を値「BBB」に変更 → Commit
> 3. ブランチ B でブランチ DD → 「Sync」（または Submit 時の自動 Sync）を実行
> 4. 同一セルの競合でコンフリクトが発生する

| # | 確認内容 | 期待結果 | 結果 |
|---|---------|---------|------|
| 9-01 | Sync 時にコンフリクト発生 | ConflictView オーバーレイが表示される | ☐ |
| 9-02 | テーブル選択 → 行ごとに Ours / Theirs 選択 | 選択が反映される | ☐ |
| 9-03 | 解決後に再コミット | コンフリクトが解消され、通常 Idle 状態に戻る | ☐ |
