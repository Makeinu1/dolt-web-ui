# Dolt Web UI — ブラウザ手動試験チェックリスト

このドキュメントはブラウザで実施する最終受け入れ試験のチェックリストです。
E2E スクリプト（APIレベル）が通過した後、UI の動作・表示・インタラクションを人手で確認します。

**前提条件:**
- `./dist/dolt-web-ui -config config.yaml` でサーバ起動済み
- Dolt SQL Server（ポート3306）が起動済み
- `Test` データベースに複数テーブルとデータが存在する状態
- ブラウザは Chrome または Firefox（最新版）
- セクション7（バージョン比較）の実施前に **比較対象のブランチが2つ以上** 存在すること
- セクション8（マージログ）の実施前に Approve を **1回以上** 実施済みであること
- セクション9（コンフリクト解決）の実施前に意図的なコンフリクトを作成しておくこと（手順は各セクション冒頭に記載）

**テスト実行順序の注意:**
セクション1→2→3→...の順に実施してください。各セクション開始時点でワークブランチが存在することを前提とします（ブランチ削除テストはセクション16に集約）。
